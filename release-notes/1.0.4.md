# Solem Toolkit 1.0.4

## What's changed

- Fixed manual station command sequencing for BL-IP controllers by waiting for controller notifications during the initial handshake, command write, and commit phases.
- Preferred Home Assistant Bluetooth registry resolution before falling back to direct Bleak scanning.
- Added reverse-engineering reference material under `_reverse_engineering/` to document the protocol used by this fork.

## Why this release

This release is aimed at the manual station start issue where station 1 could work but other stations would fail to start reliably. The BLE pacing now follows the verified controller flow more closely.

## Upgrade notes

- Restart Home Assistant after updating the integration.
- If manual station control still fails, enable debug logging for `custom_components.solem_toolkit` and capture the BLE notification sequence.
